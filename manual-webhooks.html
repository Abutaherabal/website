Setup without granting admin access

Using Greenkeeper with an out of the box setup requires you to grant the service admin access to your GitHub repositories. We require this soley so we can setup webhooks for you – an integral part of what makes Greenkeeper work the way it does.

Granting access and letting us do the setup for you has some clear advantages.

  1. We can make sure everything is setup correctly
  2. In case the setup needs to change we can migrate on our own
  3. You don’t have to think about or do any of this

While trying our best to be as trustworthy as possible we realize that admin access is something many people can not or do not want to grant a third party like us. We’ve got you covered.

If you want Greenkeeper to operate on private and public repositories this is how it works. Requires greenkeeper@2.13.0 or later.

greenkeeper signup --force --no-admin --private

If you only need Greenkeeper for public repositories you can further reduce the scope.

greenkeeper signup --force --no-admin

Once the signup is completed you can enable repositories as usual. Detailed instructions on how to setup the hooks will follow after you have enabled a repository.

greenkeeper enable --slug user/repo

Please note that you still need admin access to a repository to setup Greenkeeper. The only difference is that you don’t need to grant that access to us.

Setting up the webhooks

Great, you made it! You enabled without granting us admin access for it. Following this link to the GitHub settings page of your repository you have to enter the following configuration settings.

Payload URL

https://api.greenkeeper.io/webhooks/github

Content type

application/json

Secret

Which events would you like to trigger this webhook?

Let me select individual events.
  • Pull request
  • Push
  • Repository
  • Status

Active

And that’s it. As long as you have entered the configuration exactly as shown Greenkeeper will work without admin access.
